Extracting from Packages

If all you need are the resources, rather than the package itself, then you can extract them and forego the Package object.

Extracting Resources

The static extractResources(buffer) method accepts a DBPF buffer as an argument, parses it, and returns a list of ResourceKeyPair objects, which have key and value properties.

These key/value pairs are just like entries, except they do not belong to any mapped model.

Optional Arguments

The extractResources(buffer, options?) method accepts a second argument, which is an object of PackageFileReadingOptions.

One of the most useful options is resourceFilter, which lets you include or exclude resources based on their resource key. For instance, you can choose to extract string tables only, and ignore all other file types.

You can read through the other options if you'd like, but don't worry about it if you don't want to. Chances are, you're never going to need the other options.

Final Notes

Obviously, the main purpose of extraction is to extract tuning from the game. For that, you can use @s4tk/extraction.

It is impossible to demo game file extraction in the browser for two reasons:

  • The browser does not have access to your local file system
  • @s4tk/extraction is not just JavaScript, it uses some C bindings to optimize the extraction process

If you want to use S4TK for extracting files from your game, check out the extraction template project on GitHub.
