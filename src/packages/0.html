Package

When dealing with packages, you'll need to use the Package class from @s4tk/models.

This tutorial assumes that you are familiar with ResourceKeys, MappedModels, and @s4tk/hashing. If you are not, you should read the Intro to S4TK and Hashing tutorials before this one.

Creating a Package

To create a new, empty package, you can use new Package().

If you want to create a package with existing entries, you can pass a list of key/resource pairs as the first argument to the constructor. Check the docs if you want more information.

Tip: Never use the name "package" for a variable, as package is a reserved word in JavaScript - I usually stick with "pkg".

Adding Entries

To add an entry, you need a resource and a resource key. A resource key is an object with type, group, and instance properties.

Once you have a resource and its key, you can add them to the package with the add(key: ResourceKey, value: Resource) method.

Getting Entries

Since a package is a mapped model, each key/value pair is called an entry. Specifically, the entries in a package are of the type ResourceEntry, which has id, key, and value properties.

You can get a resource entry by its...

  • ID: get(id: number)
  • Key: getByKey(key: ResourceKey)
  • Value: getByValue(value: Resource)

Additionally, you can use the entries property, which is a list of all of the entries in the package. You can use this list to get entries by their index, or to loop/iterate over all of the entries.

Note that adding/removing entries to/from the entries list will NOT add/remove them from the package - you MUST use the add/delete methods.

Editing Entries

Once you have a ResourceEntry object, editing it is as easy as editing the key and value properties.

You can either set these to entirely new keys and resources, or you can mutate the existing key/resource.

Note that resource entries also have a property called resource, which is just an alias for value.

Deleting Entries

You can delete a resource entry by its...

  • ID: delete(id: number)
  • Key: deleteByKey(key: ResourceKey)

If you're confused about IDs vs. keys, it is explained in the Intro to S4TK tutorial.

For posterity, the key is what actually gets written to the package, and the ID is a unique identifier that S4TK uses internally (just in case any keys are repeated).
