String Table Binaries

If you want to load an existing STBL into a StringTableResource, or want to write one to a file, you need to work with binary STBL buffers.

Reading a STBL Binary

First, you have to get your hands on a buffer that contains binary STBL data. Then, you can use StringTableResource.from(buffer) to load it into a string table model.

In the sandbox, you can use await Sandbox.import(filename) to load a media file. This tutorial has an included media file called "Sample.stbl" that contains binary STBL data.

Writing a STBL Binary

To save a StringTableResource object as a file, you have to get its buffer with the getBuffer() method.

In the sandbox, you can use Sandbox.download(filename, buffer) to save the string table to your computer.

Note about Node

When using S4TK in Node, remember that import() and download() are not available. You'll have to use the fs Node module to read & write files.
