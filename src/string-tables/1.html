String Table Binaries

In order for the StringTableResource model to be useful at all, you need to be able to load one from a STBL binary, as well as write it as one. Fortunately, this is extremely easy.

Reading a STBL Binary

Before you can create a StringTableResource object, you have to get your hands on a buffer that contains binary STBL data.

In the sandbox, you can use await Sandbox.import(filename) to load a media file. This tutorial has an included media file called "Sample.stbl" that contains binary STBL data.

Writing a STBL Binary

To save a StringTableResource object as a file, you have to get its buffer with the getBuffer() method. If you view the S4TK documentation, you'll see that this method has some optional arguments, but for the most part, you don't have to worry about them.

In the sandbox, you can use Sandbox.download(filename, buffer) to save the string table to your computer.

Note about Node

When using S4TK in Node, remember that import() and download() are not available. You'll have to use the fs Node module to read & write files.
