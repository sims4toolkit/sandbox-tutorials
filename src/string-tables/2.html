String Table JSONs

Using JSON for string tables is a bit niche, but when done correctly, can make your string management much easier.

Reading a STBL JSON

The StringTableResource constructor is able to accept a list of objects containing key/value pairs - this is a JSON. This JSON can either be defined as an array literal, or it can be loaded from a JSON file.

Regardless of how you get your hands on the JSON, all you have to do is pass it into the constructor, and you'll have a string table with default entries.

You can add, get, edit, and delete entries from these string tables just as you did with the string table you created from scratch on the previous page.

Writing a STBL JSON

Once you have a StringTableResource, regardless of how it was made, you can turn it into a JSON with the toJsonObject() method.

From there, you can do whatever you want with the JSON. Iterate over it, save it to a file, load it into another STBL - anything.

The toJsonObject() method accepts some optional arguments. If you want to learn about them, check out the documentation.
