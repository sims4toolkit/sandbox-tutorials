StringTableResource

When dealing with string tables, you'll need to use the StringTableResource class from @s4tk/models.

This tutorial assumes that you are somewhat familiar with mapped models. If you are not, you should read the Intro to S4TK tutorial before this one.

Creating STBL

To create a new, empty string table, you can use new StringTableResource().

Adding Strings

The addAndHash(value: string) method adds a string to the table, using its FNV32 hash as the key. However, hashing the value itself is not very safe in terms of making a unique hash.

Instead, you should use addAndHash(value: string, toHash: string), which hashes the toHash argument for the key instead. This is safer than hashing the value itself, because you can append your creator name to the front.

Finally, if you know the exact key you want, you can use add(key: number, value: string), which adds the key/string pair to the table.

Getting Strings

Since a string table is a mapped model, each key/value pair is called an entry. Specifically, the entries in a string table are of the type StringEntry, which has id, key, and value properties.

You can get a string entry by its...

  • ID: get(id: number)
  • Key: getByKey(key: number)
  • Value: getByValue(value: string)

Additionally, you can use the entries property, which is a list of all of the entries in the string table. You can use this list to get entries by their index, or to loop/iterate over all of the entries.

Note that adding/removing entries to/from the entries list will NOT add/remove them from the string table - you MUST use the add/delete methods.

Editing Strings

Once you have a StringEntry object, editing it is as easy as setting the key and value properties.

Note that string entries also have a property called string, which is just an alias for value.

Deleting Strings

You can delete a string entry by its...

  • ID: delete(id: number)
  • Key: deleteByKey(key: number)

If you're confused about IDs vs. keys, it is explained in the Intro to S4TK tutorial.

For posterity, the key is what actually gets written to the string table, and the ID is a unique identifier that S4TK uses internally (just in case any keys are repeated).
