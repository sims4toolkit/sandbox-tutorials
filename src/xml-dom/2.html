Value & Comment Nodes

Value nodes are much simpler than element nodes. They are just a single value, such as a string, number, or boolean. Comments are even easier, as they are just a string.

Parsing Values

Like attributes of elements, the XML parser will always load the value of an XmlValueNode as a string, even if it's a number. This is again due to precision errors with 64 bit numbers. If you need the value as a number, you must parse it with BigInt().

Getting & Setting Values

You can get or set the value of both XmlValueNodes and XmlCommentNodes with their value property.
