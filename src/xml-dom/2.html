Value & Comment Nodes

Value nodes are much simpler than element nodes. They are just a single value, such as a string, number, or boolean. Comments are even easier, as they are just a string.

Getting & Setting Values

You can get or set the value of both XmlValueNodes and XmlCommentNodes with their value property.

If you have an element node and would like to get the value of the value/comment node within it, you can use innerValue (an alias for child.value).

Setting value to strings, numbers, and bigints should work as you expect, but keep in mind that booleans will write with a capitalized first letter (True and False).

Note on Parsed Values

Like the attributes of elements, the XML parser always loads the value of an XmlValueNode as a string, even if it's a number. This is again due to precision errors with 64 bit numbers. If you need the value as a number, you must parse it with BigInt().
