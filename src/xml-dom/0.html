About XML DOMs

As a TS4 modder, you're likely familiar with XML structure. However, in order to work with XML in a script, you must also understand XML DOMs.

XML is a data format that means nothing on its own. In order for it to have meaning, it must be loaded into a Document Object Model (DOM). The S4TK module that handles XML DOMs is called @s4tk/xml-dom.

Nodes

At the core of XML DOMs are nodes. "Node" is a vague term for pretty much anything in an XML DOM, such as a tag (like <T />), a value (like 0x00000000), or a comment (like <!--...-->).

Some nodes, like tags, are able to contain other nodes. These are called parents, and the nodes they contain are their children. For example, an <L /> tag that contains several <T /> tags is a parent, and the <T /> tags are its children.

Everything in an XML DOM is a node, including the document itself. The document node is just a wrapper for the nodes that do not have a visible parent. In properly structured XML, there should only be one* node without a parent, such as the <I /> in instance tuning or <M /> in module tuning. This one child is called the root.

* You may see one more node - the line that begins with "<?xml". This is called a processing instruction, and can be ignored for the sake of this lesson.

XML Documents

If the last section was too abstract for you to digest, take a look at the script editor for a real example. In the XML document created for the doc variable, <I> is the root node. It is a parent with one child, <L>. <L> is also a parent to the two <U> nodes, and each <U> is a parent to a <T> and an <E>. Even those <T> and <E> nodes are parents - their children are the value and comment nodes they contain, but we'll get back to that later.

You'll notice that the XML string is being parsed into a DOM with XmlDocumentNode.from(xml: string | Buffer). This static method returns an instance of the XmlDocumentNode class, which represents an XML document.

As mentioned before, the root of this document is <I>, as it is the node from which everything else descends. To access it, you can use the doc's child property (the children property is a list of all of a node's children, and child is just an alias for children[0]).

Attributes

Attributes are the key/value pairs that appear within opening tags, such as the n="tunable" in <T n="tunable">.

We'll touch more on attributes on the next page, but for now, just know that you can access a tag's attributes with its attributes property. Once you have that, you can just use the attribute's key to get its value, like attributes.n or attributes["n"] (these do the same thing).
