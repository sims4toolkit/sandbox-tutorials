Methods

Other than manually getting, setting, and adding parts of the DOM, there are some helper methods that you may find useful.

Finding Children

So far, you've been getting child nodes by their index. However, this is pretty impractical, since you'll rarely know what the index is, and it is likely to change whenever you update your file anyways.

That's where findChild(name: string) comes in. It will search the children list for the first child that has the given name.

Sorting Children

Keeping your tunables in alphabetical order by name is important for readability, but can be tedious when you're programmatically adding children to the end of the list.

And for this, we have sort(). This method can optionally accept a function to use for sorting, but you're unlikely to need it - by default, it will sort children alphabetically by their n attribute.

Writing XML Documents

You've seen the toXml() method a few times now, but haven't been told what it does. Simply put, it takes your XML DOM and turns it into a string, so that you can log it, write it to a file, or save it in a package. Admittedly, a better name would be toString(), but hindsight is 20/20.
