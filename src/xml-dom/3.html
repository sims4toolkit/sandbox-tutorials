Creating New Nodes

So far, you've seen how to parse an XML document and how to get/set individual values within nodes. This page will show you how to create and add new nodes.

Using Constructors

Each node type has its own respective constructor. You can check the documentation linked above for more information about the expected arguments, and the script editor for examples of them being used.

To actually add the new nodes after you create them with their constructors, you can use the node's addChildren() method, which accepts an arbitrary number of nodes to add to the end of the children list.

Using @s4tk/tunables

There is an S4TK module that exists for the sole purpose of making this process easier, it is called @s4tk/tunables.

This module contains helper functions that make your code more readible and require less arguments. For instance, compare lines 32-41 with lines 69-73 in the script editor.

Using Clones

You can clone nodes with the clone() method. This will return a deep copy of the node, meaning that it and all of its children are new instances with the same values as before.

Once you have a clone, you can edit it. This is particularly handy when dealing with lists, since their children tend to have similar, if not the same, structure.
