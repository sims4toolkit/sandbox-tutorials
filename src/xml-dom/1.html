Attributes

As mentioned on the last page, attributes are key/value pairs. In tuning, they appear on the root node, its children, children of tuples, and variants.

Attribute Aliases

On the previous page, you read that you can access attributes with the attributes property. However, for some common attributes (n, s, and t), there are aliases available.

To get the n attribute of a node, you can use name. For s, you can use id. And finally, for t, you can use type.

For any other attributes (like c, m, i, and p), you must use the attributes property and get them by name. These attributes do not have aliases for two reasons:

  1. You're unlikely to need them.
  2. There are no good options*.

*class/module are reserved words and cannot be used for c/m; instanceType is too long and type is taken by t, neither is good option for i; who the hell cares about p?

Editing Attributes

To add a new attribute or change the value of an existing one, all you have to do is set it, like node.attributes.n = "something". To delete an existing attribute, you need to use delete node.attributes.n.

Valid values for attributes include strings, numbers, and bigints. Note that parsing an XML document will always return them as strings, because parsing them as numbers would cause precision errors with 64-bit tuning IDs. If you need the instance to be a bigint, you must cast it with BigInt().

It's important to note that you are not allowed to set the attributes property itself to a new object. You must mutate the existing one*.

* This is a remnant from how S4TK used to handle buffer cacheing, and may be changed in the future. But, for now, attempting to set attributes to a new object will throw an error, so don't do it.
