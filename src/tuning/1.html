The XML DOM

Assuming its content is valid XML, you can access an XML resource's DOM with its dom property.

Accessing The DOM

The DOM is an XmlDocumentNode, and can be get/set* with the dom property. Since you're almost always going to want the DOM's child, the root, you can also use the root property (an alias for dom.child).

It's important to note that editing the DOM does NOT automatically update the content. And, since the content is the source of truth, that means that edits made to the DOM will not be reflected the next time you use content or call getBuffer().

* When setting dom/root, note that the content property gets automatically reset to the value of dom.toXml().

Editing The DOM

Since editing the DOM does not automatically update the content, you have two options:

  • Set the dom property equal to itself after you make your edits, which will then set content equal to dom.toXml()*
  • Edit the DOM using the updateDom() / updateRoot() methods, which automatically update the content when they finish

While there's nothing wrong with setting the DOM to itself, it is less readible and more prone to error (e.g. forgetting what it's for and deleting it, or forgetting to do it in the first place).

So, the second approach (using the updateDom() / updateRoot() methods) is preferred, and it's good to get into the habit of using them right away.

* You may be thinking that content = dom.toXml() is a third option, but it is strongly discouraged, as doing so would delete the DOM and require it to be regenerated.
