XmlResource

When dealing with XML resources (like tuning or ASMs), you'll need to use the XmlResource class from @s4tk/models.

This tutorial assumes that you are familiar with @s4tk/xml-dom. If you are not, you should read the XML DOMs tutorial before this one.

Creating New Tuning

To create a new, empty tuning file, you can use new XmlResource() with no arguments.

If you have initial content to use, you can pass either a string or an XmlDocumentNode (from @s4tk/xml-dom) as an argument to the constructor.

Loading Tuning From File

Like all other resources, the XmlResource class has a static from(buffer: Buffer) method that can be used to load an existing tuning file into an XmlResource object.

This tutorial supplies a sample tuning file called "Sample.xml", and you can load it as a buffer with Sandbox.import().

The Content

The "content" of an XML resource is a string that contains its XML as plain text. You can get/set* it with the content property.

The content is the source of truth for XML resources. That is, it is used in getBuffer(), so whatever the content is is what gets written to files and packages.

* Setting content will regenerate the DOM. There is a somewhat complex relationship between the content and DOM. This will be discussed further on the next page.
