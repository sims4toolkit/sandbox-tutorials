Keys & Enums

You'll be working with resource keys a lot, and S4TK has several enums to help you with them. While you don't need to know everything about these enums just yet, you should be familiar with which ones are used for what.

Types

For tuning types like actions and snippets, you can use the TuningResourceType enum. For binary file types like string tables and SimData, you can use the BinaryResourceType enum.

The reason tuning and binary types are split into two enums is so you can use type in TuningResourceType to determine if a resource is tuning based on its type.

Groups

For most resources, the group number does not matter and should be set to 0. However, for SimData, the group serves as a secondary type, and must be the 24-bit version of its associated tuning's type. For SimData groups, you can use the SimDataGroup enum.

Not every tuning type has an associated SimData group, and not every SimData group is the 24-bit version of its 32-bit tuning type. These are general patterns, but there are exceptions.

Instances

Instances are the unique part of resource keys, and there is therefore no enum for them. However, the top byte of string table instances must match their locale, and there is an enum for that - StringTableLocale.

The StringTableLocale enum also has some helper functions that will set the high byte of an instance for you. This is briefly demoed in the script editor, but you'll read more about it in the string table tutorial.

Hashing

Another very important aspect of resource keys is FNV hashing. If you want to learn how to generate FNV hashes using S4TK, you should check out the very brief hashing tutorial when you are done with this one.
