S4TK Models

You're going to learn about several S4TK models, but it may help you to know about some common patterns that appear throughout the entire library.

Writable Models

While this isn't something you'd normally have to know about, all of the models in S4TK inherit from a common base class called WritableModel. This is just worth mentioning because it helps you understand what the models have in common with each other.

One of the most important commonality between writiable models is their static from() method. Every class that inherits from WritableModel implements a static from(buffer: Buffer) method that accepts a single buffer as an argument, and may accept optional arguments in an object as a second argument.

You're going to see this from() method used in every model tutorial, so it's good to be aware of.

Mapped Models

MappedModel is another anbstract base class that you generally don't need to worry about, but again, it's worth mentioning so you have context for the models that inherit from it.

A MappedModel inherits from WritableModel, so everything said in the last section also applies here.

At time of writing, there are two mapped models: Package and StringTableResource. What mapped models have in common is that they map keys to values. In the case of packages, they map resource keys (type, group, instance) to resources, and in the case of string tables, they map 32-bit integers to strings.

The vast majority of methods that are used to interact with mapped models appear on the MappedModels base class, so be sure to review that documention when working with packages and string tables.

This also means that once you've mastered working with one of them, you've mastered working with both, since they're essentially the same thing just with different types for their keys and values.
