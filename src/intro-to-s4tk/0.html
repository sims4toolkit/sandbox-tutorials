Welcome to S4TK

S4TK's tagline is that it can "read and write package files and the resources they contain", but what does that mean?

Terminology

Packages are just a data format, somewhat similar to a ZIP archive, and mean nothing on their own. What really matters are the resources they contain, such as tuning, string tables, SimData, object definitions, etc.

S4TK is driven by models, which are just classes that represent data. S4TK has models for packages and resources, and they are all defined in the @s4tk/models npm package.

S4TK models can be read from and written to buffers, which are fixed-length lists of bytes containing data. This data can either be binary, like packages, or plain text, like XML.

Do not worry - you don't need to understand binary file formats or how buffers work to use S4TK. Using the fs Node module, you can read/write buffers from/to files.

If you'd like to learn more about buffers, check out the documentation for Node's Buffer class.
