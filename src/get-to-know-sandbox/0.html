User Interface

Welcome to the S4TK Sandbox. This website is intended to make S4TK more accessible by removing the need to set up your own Node environment.

In order to get the best out of this website, you should understand how it works. That's what this tutorial is here for.

Main Editor vs. Tutorials

There are two primary features of this website. The first is where you are now, in an interactive tutorial. The second is the main code editor, in which you have access to a file system and can write, save, and run any scripts you want.

To access the main code editor (once you're done with this tutorial), click the button in the navbar. To come back to tutorials, click .

For now, don't worry about the main code editor and file system - they'll be mentioned as needed throughout this tutorial, and you'll be prepared to use them by the end.

Tutorial Guide

The tutorial guide is what you are reading right now. It is only available in tutorial mode, and is replaced with a file manager in the main editor. You can resize this window by dragging the edge between it and the script editor.

Some tutorials, such as this one, consist of multiple pages. You go back and forth, you can use the Back () and Next () buttons at the bottom of the tutorial guide.

The Script Editor

The script editor is the window to the right of this one. You can type, copy/paste, undo/redo, and even find/replace using your computer's standard keyboard shortcuts.

You can edit the content to whatever you want, but you cannot save these edits while in tutorial mode - if you leave/refresh this page, the editor content will reset.

When using the main editor/file system, you can save your changes, and they will be here waiting for you if you leave the site and come back. In fact, you can save as many scripts as you'd like - that's what the file system is for.

The Console

The console is the window below the editor. You can resize it by dragging the edge between it and the script editor.

Its purpose is to show the output generated by your script. Errors will automatically be logged in the console, but there are ways you can write to it yourself.

Sandbox.output(...args: string[]) will write the arguments it is given separated by newlines. Sandbox.test(name: string, condition: boolean) will check the value of the condition and print out a pass/fail message accordingly.

The Toolbar

The toolbar is the set of buttons in the bottom-right of your screen. In tutorial mode, these buttons include:

Download () will download the contents of the editor as a JavaScript (.js) file.

Reset () will reset the contents of the editor to how they were at the start of the tutorial.

Run () will run the contents of the editor as a script and write its output to the console.
