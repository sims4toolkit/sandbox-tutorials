Node Parity

Although the sandbox uses browser JavaScript, it is intended to mock a Node environment. As such, the sandbox provides implementations for some Node features, such as require() and Buffer.

Importing S4TK Modules

You can import S4TK modules exactly as you would in Node, using require(). Imports in this sandbox are at full parity with the S4TK Node library, so you can copy your code directly into Node and it should "just work" (minus the Sandbox functions).

However, since this is a browser, you do NOT have access to any Node modules other than S4TK. This includes fs, the Node file system. When converting S4TK scripts from the sandbox to Node, you'll have to replace import() with readFileSync() and download() with writeFileSync().

Buffers

Buffers are very important in S4TK, as they are used to read and write files. The buffers returned from import() and resources' getBuffer() methods are at full parity with buffers in Node.

In fact, if you need to use the Buffer class (you probably don't, but in case you do), you can use it just as you would in Node - no import required.

Final Notes

If you see code in this script that you don't understand, such as StringTableResource and formatStringKey(), don't worry about it. These are explained in other tutorials.
