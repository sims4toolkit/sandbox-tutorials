Writing & Running Scripts

Now that you're familiar with the sandbox interface, functions, and Node compatibility, you're almost ready to get started with the main editor and file system.

But, before you go clicking that button, here are a couple of final notes.

Saving Scripts

For performance reasons, the script does not autosave as you type. When you're in the main editor, you'll see an indicator at the top of your file letting you know when you have unsaved changes.

To save your script, you can press the Save () button in the toolbar (only available in the main editor). Alternatively, your script will also be saved every time you download or run your script, or when you switch between files.

If you close the website without saving your changes, you will lose them.

Running Scripts in Scripts

The scripts you have saved in your file system are allowed to call each other by name, and can even pass values to one another. Since each individual script is wrapped in an asynchronous function, you can use the return keyword in one script, and await when calling it in another.

Tutorials do NOT have access to the file system, and therefore cannot run other scripts. If you run the script for this page, you'll see an error.

To call a script from another script, use Sandbox.runScript(filename: string). Like import(), this is an async function, so you must use the await keyword in order to get its value.

You're Done!

It's time to check out the main editor and file system - click in the navbar to get there.

Once in the main editor, you'll see an example script that demonstrates everything you'd read about in this tutorial. The script "helloWorld" gets a string table from the "loadSampleSTBL" script, which reads the "SampleSTBL.stbl" media file into a StringTableResource object.
