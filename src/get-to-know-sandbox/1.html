Sandbox Functions

The sandbox has several built-in functions, such as output() and test() which were mentioned on the previous page. This page will teach you about the other sandbox functions.

The File System

In the file system, there are two types of files: scripts and media. Scripts are the JavaScript files that you write & run, and media files can contain any data you want, such as JSON, string tables, or packages. Scripts can import media files and use them however they please.

Tutorials do not have access to the main sandbox's file system, but some of them provide files of their own. This tutorial provides a media file called "Sample.stbl", which is a string table containing one entry.

Importing Media Files

You can import media files with Sandbox.import(filename: string), which is an async function that returns a Buffer. The scripts you write are executed in the context of an async function, which means you are allowed to use top-level await.

If that last paragraph went over your head, don't worry about it - just look at how the import() function is used in the sample script and use it like that. Note that using await is very important - if you forget it, the import function will not return the value you're expecting it to.

Downloading Files

You can download anything you want using the Sandbox.download(filename: string, content: string | Buffer) function.

The download() function will not download your file immediately, but rather it will queue it to be downloaded when the script is done. If only one file is queued, it will download as-is. If multiple files are queued, they will be zipped and then downloaded as one. This means that you can use download() multiple times, or even in a loop, and you don't have to worry about it cluttering your downloads folder.

After running this script, a file called "Get_to_Know_the_Sandbox.downloads.zip" should automatically download, and it should contain two files: "MyStbl.stbl" and "MyText.txt".
