When dealing with string tables, you'll need to use the StringTableResource class from @s4tk/models.

Creating

To create a new, empty string table, you can use new StringTableResource().

To create a new string table that contains known strings, you can use new StringTableResource(jsonData), where the jsonData argument is a list of key/value pairs.

Alternatively, if you have a buffer that contains binary STBL data, you can use the static StringTableResource.from(buffer) method. This will read the binary data into a new StringTableResource object, and return it.

Editing

The addAndHash(string) method will add the given string to the table, and use its 32-bit hash as its key.

The addAndHash(string, toHash) method will add the given string to the table, and use the 32-bit hash of the toHash argument as its key.

The add(key, string) method will add the given string to the table with the given key.

Writing

In order to save a string table, you need to get its buffer. You can do this with the getBuffer() method.

In this sandbox environment, you can use Sandbox.download() to download your string table. In a Node environment, you'd use the fs module, and save your file withfs.writeFileSync("path/to/file", stbl.getBuffer()).
